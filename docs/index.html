How this works

Most note archives end up scattered: a title here, a tag cloud there, a wall of text somewhere else. This page keeps the vault flat and lets the metadata do the organization.

Each note is stored as Markdown at the repo root. The reader pulls frontmatter, builds the archive, and keeps the note itself as the source of truth. Search, tags, and related links are the navigation layer.
